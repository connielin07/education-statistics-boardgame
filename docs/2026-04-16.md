# 2026-04-16 專案檔案功能盤點

本文件依照目前專案內容掃描整理，說明各檔案目前負責的功能、是否已接到主流程，以及需要注意的地方。

## 目前主流程

目前網頁入口是 `index.html`，載入 `js/app-init.js`。`app-init.js` 在 `DOMContentLoaded` 後渲染首頁、遊戲頁、結果頁與 RWD 提示，並透過 `pageFlow.js` 控制首頁、遊戲頁、結果頁的顯示切換。

實際遊戲主流程目前集中在 `js/home-info-result/infoView.js`：

1. 從 `data-source/export/schoolData.js` 隨機抽 3 所學校。
2. 顯示學校資料、資源分配按鈕、事件按鈕、回合與分數。
3. 玩家分配剛好 3 點資源後按 FINISH。
4. 呼叫 `js/resource-and-score/scoreRule.js` 的 `calculateRoundScore()` 計算本回合分數。
5. 用 `updateTotalScore()` 更新累積總分。
6. 三回合後觸發結果頁，結果頁從 `state.allocationHistory` 與 `state.totalScore` 顯示各回合分數與總分。

## 需要注意的整合點

1. `infoView.js` 目前包辦抽學校、卡片渲染、資源操作、回合推進、計分與結果頁切換，功能集中度高。
2. `allocationUIController.js`、`resourceAction.js`、`ruleCheck.js` 是較模組化的舊設計，但現行主流程未完整採用。
3. `eventData.js` 已存在，但目前遊戲事件卡在 `infoView.js` 中是固定的 `交通不便加劇`，尚未隨機抽事件卡。
4. `drawCard.js`、`schoolCardView.js`、`eventCardView.js` 尚未實作；如果之後要按原分工拆模組，可把 `infoView.js` 內的抽卡與卡片 markup 拆到這些檔案。
5. `base/reset.css` 內容目前不是 reset，而是結果頁樣式。因為 `result.css` 後載入，實際結果頁多半以 `result.css` 為準，但檔名與內容不一致，後續維護容易混淆。
6. 文件中的部分進度與設計仍停留在 2026/04/11，和目前程式碼狀態已有差異；後續若要交給組員，建議以本文件更新後的主流程描述為準。
